Building Spark Streaming App - Part 1 (Introduction to Business Case)

08 Apr 2016

Spark Streaming is the most widely used framework to develop near real time applications in present times. It comes with lot of advantages compared to its competeting streaming frameworks available in open source world.

I am starting a new blog series in which I will be discussing about development of real-time application in Spark Streaming and various aspects of Streaming application, starting from high level Business use case of client to in-depth coding samples.

Whenever we think of any software application it is obvious that we cannot achieve every requirement with one single software tool. In the similar grounds, for our streaming application we will not only look into the spark streaming framework, we will also sneak a peek into other frameworks like Apache Kafka,Apache Cassandra and Apache Zeppelin.

Before starting of the series, I would like to mention about audience . Since this is a real application development it is good if you have prior knowledge about basics of Spark Streaming. A basic idea about streaming applications also would be helpful to get better understanding.

In this first blog I will be discussing about what is the real use case we are dealing in this series. The problem we are trying to solve can be called as Real Time log analysis.

Existing Business

Our client is a large Wi-Fi hotspot provider and installer in public spaces across US. Since it is a consumer oriented business they need to have a good application which can give valuable insights about users of their hotspot devices. Initially it was started as a batch system where data is collected for one full day and analysis is run overnight to get insights on next day.

Problems in Existing System

  1. Batch system results was not much helpful to make decisions for the business
  2. As more and more hot spots are getting installed and company started to expand the existing analytical system was not able to handle the large data loads
  3. Number of users using the hotspot for internet connectivity started to decrease over a period of time, and lot of users failed to login to the Wi-Fi network.

New system needs

Now itâ€™s obvious that there is a need for real time analytical engine which can give better idea of the system rather than having a batch system which provides analytics in a delay of one day. For this kind of real-time/streaming systems Spark Streaming is the best choice to accomplish the requirements.

Along with the ability to get the insights of the system in real time there is a big need of reporting system which can show the results to client in real time. To accomplish the reporting part of the requirement we will be using Zeppelin.

Why Spark Streaming ??

There are many streaming frameworks available to perform real time analytics. But why we have to go with spark streaming.

  1. Using spark streaming it is very easy to port a batch system into streaming system.
  2. Since spark streaming is an extension of Spark batch API, we can always provide historical data analytics in future if needed.
  3. Spark has excellent community and support in integrations for both input data sources and output reporting engines.
