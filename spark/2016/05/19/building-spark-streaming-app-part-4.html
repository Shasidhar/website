Building Spark Streaming App - Part 4 (Zeppelin Visualization)

19 May 2016

In previous blog we discussed about how we populate the results in output files. As I mentioned in last blog , we will be discussing about how we show the results to the business user. In my first blog of the series we decided to use Zepplin as an interface to show end results. So, lets see how we show results in Zepplin.

Zeppelin Dataloading

First step in using Zeppelin is to make zeppelin fetch data from our text files. Actually there is no special exercise we need to do it, we can directly use the normal spark way of loading the data and create RDDs or Dataframes inside zeppelin directly. One good thing about zeppelin is, once we start zeppelin, we get spark context created by default for us, it is similar to how we get Spark Context when we start spark shell.

In our case we have data in csv format, we will need to read RDD and create Dataframes so that we can directly write queries and get the required results. So, i will show creation of Dataframe for country stats here.


  case class countryStats(date:Date,hour:Int,country:String,count:Int)

  val countryStatDF =  sc.cassandraTable("sensoranalytics","countrystats").map(row=>{
      val cal = Calendar.getInstance()
      cal.setTimeInMillis(DateTimeFormat.forPattern(pattern).parseDateTime(row.get[String]("date")).getMillis)
      countryStats(new Date(cal.getTimeInMillis),cal.get(Calendar.HOUR_OF_DAY),row.get[String]("country"),row.get[Int]("count"))
    }).toDF

If you see the code, we have a case class for countrystats first and then normal transformation to convert RDD into Dataframe.Similiarly we will be having Dataframes for other result sets also.

Zeppelin for visualization

Lets start with what this zeppelin is all about first.Zeppelin provides notebook based web interface for apache spark framework. Its very powerful when we need to publish and share the reports across different people. Along with spark zeppelin supports other frameworks also like cassandra, flink etc. Its very easy to setup and start using it. In our case we will be doing native spark coding inside zeppelin to show the results in different charts.

Our output has hourly aggregated results. Lets look into the format of the result we generated. Each row in a counrty wise aggregate is having (2016-03-26 07:00:00,India,2) format. We know in Zeppelin we want different grouping that is hourly, daily, weekly and monthly. Lets see how we achieve each of them.

Recent Hours Aggregation

In this aggregate we want to see the report of last given recent hours, means user will give input of how many hour of report he is interested in.

In order to achieve this we want UDF to validate the record belongs to present date and returns all the records fitting inside the recent hour provided by the user. Following is the UDF which will do the required filtering.


 def recentHours(hour:Int,inputDate:Date,input:Int):Boolean=
  {
	val cal = Calendar.getInstance()
	cal.setTimeInMillis(System.currentTimeMillis())
	val presentHour = cal.get(Calendar.HOUR_OF_DAY)
	val minHours = presentHour - input
	val presentDate = new Date(cal.getTimeInMillis)
	if(DateUtils.isSameDay(presentDate,inputDate)){
	 if(hour<=presentHour && hour>=minHours){
	  true
	   }else{
		false
	   }
	 }else{
		false
	 }
  }

The query to get the hourly aggregate looks as follows


 sqlContext.sql(s"""select country,sum(count) from countrystats where validHours($recentHours,date,hour) 
   group by country""").map(output=>{
	 output(0)+"\t"+output(1)
 })

If you observe in the above query we have a placeholder $recentHours where user can give input. This denotes the number of recent hours user is interested to get results.

Hourly aggregation for a day

In this aggregate we want to show the count of an aggregate grouped by hours for a given day. As this is a straight forward problem we can directly use sql capabilites to show the results. The query looks as shown below.


  %sql select hour,country,sum(count) as count from countrystats where date=${Input} group by hour,country

Here also we have a place holder ${Input} which is where user can enter date for which report will be displayed.

Weekly/Monthly aggregation

Similarly we will have two more queries for weekly and monthly aggregation.


  %sql select weekofyear(date) as week,country,sum(count) as count from countrystats group by weekofyear(date),country
  %sql select month(date) as month,country,sum(count) as count from countrystats group by month(date),country

One thing to be observed in this case is, we are using all the built-in UDFs spark provides exactly the same way directly inside Zeppelin SQL prompt. This shows that how Zeppelin along with Spark, is easy to use and yet powerful for visualizing Bigdata.
