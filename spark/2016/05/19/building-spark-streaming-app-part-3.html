Building Spark Streaming App - Part 3 (First Iteration)

19 May 2016

In previous blog we have discussed about data formats we are dealing with and the results we have to generate. In this blog we will focus on implementing the same. Let’s see the first method of implementation, i.e. Iteration 1. Going forward, we will discuss on required changes on this implementation and need for the changes. Streaming Components

In any streaming application there will be three components, i.e Input Source, processing and Output Store. Hence, in first iteration we will use following,

  1. Input Source - Files
  2. Processing - Spark Streaming APIs
  3. Output Store - Files

By looking at these components, it’s obvious that we will be using FileInput and FileOutput streams of spark streaming engine in this phase.

Before we start looking at coding for different components let’s see data structure we will be using to map input record into a scala object.

SensorRecord


 case class SensorRecord(dateTime: DateTime,
	country:String,
	state:String,
	city:String,
	sensorStatus:String)

Receiver

As I have mentioned above, we will be using FileInput Stream to read our source data .So that we can process them in spark streaming. Following code snippet shows how to create DStream from files.


	val ssc = new StreamingContext(sparkConf,Seconds(batchTime))
	val data = ssc.textFileStream(streamingConfig.getString("source-dir"))

If you observe the code snippet, there are two parameters.

  1. batchTime is the spark streaming batch interval.
  2. source-dir is the path from where we want to pull the data.

Aggregations

As we have discussed in previous blog, we are interested in generating three aggregations. Lets see Country wise statistics calculation function.


 def calculateCountryStats(parsedDstream:DStream[SensorRecord],windowInterval:Int,slideInterval:Int):DStream[String]=
  {
	val countryAggregation = parsedDstream.map(eachRec=>((eachRec.dateTime,eachRec.country),1))
	val resultStream = countryAggregation.window(Seconds(windowInterval),Seconds(slideInterval)).groupByKey()
		.mapValues(_.size)
	resultStream.map
	{
		case ((dateTime:DateTime,country:String),count:Int) => 
				dateTime.toString("yyyy-MM-dd HH:mm:ss")+","+country+","+count
	}
 }

This function calculates country wise aggregations and returns back results in csv format.

Function takes three parameters

  1. parsedDstream is the DStream of each record from input data mapped to SensorRecord object.
  2. windowInterval is the window for which aggregation is done.
  3. slideInterval is the slide time which controls slide interval inside a given window.

Function returns DStream of String, and each record in output DStream will be in csv format 2016-03-26 07:00:00,India,2, and they are self explainatory. Observe that we are having timestamps till hour.

Similarly we will have two other functions to calculate State wise statistics and City wise statistics.

Output

Finally we will store the output into files using spark streaming API .


 calculateCountryStats(parsedDstream,windowTime,slideTime).foreachRDD((outputRdd,time)=>
 {
	if(!outputRdd.isEmpty()) 
	  outputRdd.saveAsTextFile(streamingConfig.getString("output-dir")+"/countryStats/"+time.toString())
 })

This completes the spark streaming side of coding where we read data and populate the hourly aggregates into files. In next blog we will see how we show these results in zeppelin.
